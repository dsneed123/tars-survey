Hi {{ user.first_name|default:user.email }}, you signed up yesterday — time to put TARS to work. Here's what makes a great first task.

Pro tip
The more context you give, the better the result. Include acceptance criteria, relevant files, and any constraints directly in the task description.

What TARS does well:

  • ✓ Adding new features to existing codebases
  • ✓ Fixing bugs with clear reproduction steps
  • ✓ Writing tests for existing code
  • ✓ Refactoring and improving code quality
  • ✓ Integrating third-party APIs and services

Example task descriptions that work great:

"Add rate limiting to the /api/users endpoint — max 100 requests per minute per IP using Redis"
"Fix the bug in checkout.py line 142 where tax is calculated before discount is applied"
"Write pytest tests for all public methods in auth/validators.py — aim for 90% coverage"
