Getting Started

From zero to your first automated pull request in under 10 minutes. Follow these steps to sign up, connect a project, and submit your first task.

Step 1 — Create an Account

1

Sign up at TARS

Visit the Get Started page and fill in your email and company name. You'll receive a confirmation email with a link to activate your account.

2

Verify your email

Click the link in your activation email. This confirms your address and takes you to the member dashboard.

Already have an account? Sign in here.

Step 2 — Add Your First Project

1

Go to Projects

From the dashboard sidebar, click Projects, then click Add Project.

2

Enter your GitHub repository

Provide the repository in owner/repo format — for example, acme-corp/backend-api. The repository must be accessible; for private repos, you'll need to provide a GitHub personal access token or install the TARS GitHub App.

3

Set the default branch

Choose the branch TARS will branch from when creating new PRs. This is typically main or develop.

4

Save the project

Click Save Project. TARS will verify it can reach the repository and display a success message.

Private repositories: TARS needs read/write access to clone your repo and push branches. We recommend creating a dedicated deploy key or service account with Contents: read & write and Pull requests: read & write permissions.

Step 3 — Submit Your First Task

1

Open the task form

Click the + button in the bottom-right corner of the dashboard, or go to Tasks → New Task in the sidebar.

2

Select a project and describe your task

Choose the project you just added, then write a clear task title and optional details. The more context you provide, the better TARS performs.

Good task titles:

  • "Fix the 500 error when a user resets their password with an expired token"
  • "Add pagination to the /api/products/ endpoint (page size 20)"
  • "Write unit tests for the calculate_discount function in billing/utils.py"
3

Submit and watch

Click Submit Task. The task appears in your task list with a Pending status. When a worker picks it up, the status changes to In Progress and you'll receive an email notification. When the PR is ready, you'll get another notification with a direct link.

What Happens Next

Once you submit a task, TARS:

  1. Queues the task and waits for an available worker
  2. Clones your repository on a fresh worker environment
  3. Analyzes the codebase to understand context, conventions, and dependencies
  4. Implements the requested changes using Claude AI
  5. Runs your existing test suite (pytest, jest, etc.) to verify correctness
  6. Opens a pull request on GitHub with a descriptive title and summary
  7. Sends you a notification with the PR link
Review before merging. TARS is autonomous but not infallible. Always review the generated PR before merging to ensure it meets your standards and doesn't introduce unintended changes.
