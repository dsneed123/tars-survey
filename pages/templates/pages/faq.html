Getting Started

TARS supports any language that Claude AI can read and write, including Python, JavaScript, TypeScript, Go, Ruby, Java, Rust, and more. Because TARS understands your codebase from context rather than relying on hard-coded language rules, it adapts to your project's specific stack. If your language has widely available open-source examples, TARS can work with it effectively.

Yes — TARS is designed to integrate with real-world repositories, not green-field projects. It reads your existing code, tests, and configuration before writing a single line, so changes fit naturally into what's already there. Most teams are up and running with their existing repo within minutes of connecting it to TARS.

Tasks are defined in a simple YAML configuration file inside your repository, or submitted through the TARS dashboard. Each task is a plain-language description of the work to be done — no special syntax required. TARS reads the task queue continuously and picks up new items as they appear.

Operations

Every change TARS makes is delivered as a pull request — nothing is merged to your main branch without human review. If TARS produces incorrect code, your tests will catch it before a PR is opened, and your team can reject or request changes on anything that slips through. TARS treats reviewer feedback as a new task and iterates automatically.

Most well-scoped tasks — adding a feature, fixing a bug, refactoring a module — complete in two to ten minutes. Larger tasks that touch many files or require multiple test iterations may take up to 30 minutes. TARS provides real-time status updates via Discord webhook so your team always knows what's in progress.

Security & Privacy

TARS operates with the minimum permissions needed: read access to your repository and the ability to open pull requests. It never pushes directly to protected branches, never stores your source code beyond the duration of a task, and all API communication is encrypted in transit. We follow OWASP secure-coding guidelines and review all generated code for common vulnerabilities before submission.

Your code is never used to train AI models and is not shared with any third parties outside of the Claude API call required to complete each task. Repository contents are held in memory only for the duration of an active task and are purged immediately afterward. Enterprise plans include a dedicated processing environment for additional isolation.
