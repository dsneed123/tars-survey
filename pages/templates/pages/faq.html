Frequently Asked Questions

Everything you need to know about TARS

TARS supports any language that Claude AI can read and write, including Python, JavaScript, TypeScript, Go, Ruby, Java, C#, Rust, and more. Because TARS reads your codebase before making changes, it adapts to your project's specific language version, framework, and conventions automatically.

TARS operates with read and write access limited to its own clone of your repository and never touches your production environment. All changes are submitted as pull requests, so your team retains full control and can review every line before anything is merged.

Every TARS change goes through a pull request, so mistakes are caught during normal code review before they can reach production. If a PR does not meet your standards, simply close it — TARS will try again with updated instructions or you can adjust the task description and resubmit.

Tasks are defined in a simple YAML configuration file in your project's config/ directory. Each task has a title, a plain-language description, and optional constraints. TARS polls the task queue on a schedule and picks up new work automatically.

TARS is self-hosted and open-source, so there is no per-seat or per-task fee from us. You pay only for the underlying Claude API usage billed by Anthropic, which varies based on the size and complexity of each task. Most routine tasks cost a few cents in API credits.

Simple tasks such as adding a page, fixing a bug, or updating copy typically complete in two to five minutes. More complex tasks involving multiple files, new models, or significant logic changes may take ten to twenty minutes. TARS logs progress and posts a Discord notification when the pull request is ready.

TARS runs entirely on your own infrastructure and clones repositories using your own GitHub credentials. Your source code is sent to Anthropic's Claude API for processing; Anthropic's standard data-handling and privacy policies apply. No code is stored or shared by TARS itself.
