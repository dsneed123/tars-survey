About TARS

Autonomous software development, powered by Claude AI

What is TARS?

TARS is an autonomous coding system that uses Claude AI to build software end-to-end. Given a task description, TARS reads your codebase, implements the requested changes, runs your test suite, and submits a pull request — all without human intervention. It is designed to handle the full software development loop so engineering teams can focus on the work that matters most.

How it works

🔍

1. Read

TARS clones your repository and builds a deep understanding of its structure, conventions, and existing patterns before touching a single file.

⚙️

2. Implement

Claude AI writes the code, following your project's style and conventions exactly. Changes are minimal and focused — no unnecessary refactoring.

✅

3. Test

TARS runs your existing test suite and verifies the build succeeds. If something breaks, it diagnoses and fixes the issue before proceeding.

📤

4. Submit

A pull request is opened on GitHub with a clear description of what was done and why. Your team reviews and merges on their own schedule.

The team

TARS was created by a small team of engineers who believe that AI should handle the mechanical parts of software development so humans can concentrate on product thinking, architecture, and the problems that genuinely require human judgment. We operate lean, move fast, and ship continuously — with TARS helping us do it.

Vision

Software projects accumulate backlogs that never shrink. Routine features, bug fixes, and maintenance tasks pile up while engineers focus on high-priority work. Our vision is a world where that backlog is no longer a constraint — where any well-described task can be delegated to an autonomous agent that implements it correctly, safely, and fast. TARS is the first step toward that future.
